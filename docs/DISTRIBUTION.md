# AUM Core - Distribution Guide

## Overview

This repository is the **source of truth** for the AUM Core component library. It maintains a full buildable NX workspace (the `demo-app` shell) while distributing only the `libs/aum-core/` folder to application repositories via git subtree.

## Why Git Subtree (vs Git Submodules)?

| Aspect | Git Subtree (Our Choice) | Git Submodules |
|--------|--------------------------|----------------|
| **Clone & Go** | `git clone` works immediately | Requires `--recurse-submodules` or extra commands |
| **Build Tools** | NX sees `libs/aum-core` as regular code | May need special configuration |
| **Developer Learning Curve** | No special knowledge needed | Must learn submodule commands |
| **IDE Support** | Autocomplete works across libs | May need extra setup |
| **Update Process** | Manual: `npm run update-core` | `git submodule update --remote` |
| **Version Management** | All apps coordinate updates | Each app can pin its own version |

**Bottom line:** Git subtree provides a seamless developer experience at the cost of one extra command (`npm run publish-core`) when publishing changes. For an internal library with a small number of apps and coordinated update cycles, this is the right trade-off.

## Why Git Subtree (vs NPM Package)?

### NPM Package pros:
- Industry standard for library distribution
- Strict semantic versioning
- Apps control exactly when to upgrade
- Multiple versions can coexist during migration

### Why we chose Git Subtree instead:
- **Small scale**: 2–5 app repositories (NPM is better at 5+)
- **Coordinated updates**: All apps update together — no version fragmentation needed
- **NX integration**: Maintains native NX workspace structure and build caching
- **Simpler workflow**: No npm registry, publishing pipeline, or versioning overhead
- **Fast iteration**: Frequent updates without a formal release process
- **Internal only**: No plans to distribute externally

## Branch Structure

### `main` Branch
- Full NX workspace with complete project structure
- Used for all development and testing
- Contains: `apps/`, `libs/aum-core/`, `libs/modules/`, all config files

### `aum-core-dist` Branch
- **Distribution branch** — auto-generated by the publish script, never edit directly
- Contains **only** the contents of `libs/aum-core/` at root level
- Used by application repositories for pulling updates
- Contents: `ui/`, `theme/`, `utils/`, `common/`, `templates/`

## Making Changes to AUM Core

### Workflow

```bash
# 1. Make changes on the main branch
git checkout main
# Edit files inside libs/aum-core/...

# 2. Test changes in the shell app
npm run build
npm run test
npm run serve

# 3. Commit and push
git add libs/aum-core/
git commit -m "feat: your change description"
git push origin main

# 4. Publish to the distribution branch
npm run publish-core
# Or directly: ./scripts/publish-aum-core.sh

# Done! App repos can now pull your changes.
```

### The Publish Script

`scripts/publish-aum-core.sh` automates the distribution process:

1. Verifies you are on `main` with no uncommitted changes
2. Extracts `libs/aum-core/` using `git subtree split`
3. Creates/recreates the local `aum-core-dist` branch from the split commit
4. Updates `README.md` in the dist branch with the latest 3 commit messages
5. Force-pushes `aum-core-dist` to remote
6. Returns you to the `main` branch

**Run this script after every commit that touches `libs/aum-core/`.**

## Consuming Apps

Application repositories pull from the `aum-core-dist` branch.

### Initial Setup (one-time per app repo)

```bash
cd your-app-repo

# Add aum-core as a remote
git remote add aum-core https://github.com/rawatanimesh/aum-core.git
git fetch aum-core

# Pull libs/aum-core into your workspace
git subtree add --prefix=libs/aum-core aum-core aum-core-dist --squash
```

Or use the helper script from this repo (copy it across):

```bash
./scripts/initialize-core-subtree.sh
```

The script will add the remote, fetch, remove any existing `libs/aum-core/`, and re-add it as a proper subtree.

### Pulling Updates

```bash
# In your app repo
npm run update-core

# Or manually:
git subtree pull --prefix=libs/aum-core aum-core aum-core-dist --squash
```

Add this npm script to your app's `package.json`:

```json
"update-core": "./scripts/update-aum-core.sh"
```

And copy `scripts/update-aum-core.sh` from this repo into your app repo's `scripts/` folder.

## Rules

### DO:
- Make ALL `libs/aum-core/` changes in this repository on the `main` branch
- Test changes in the `demo-app` shell before publishing
- Run `npm run publish-core` after every commit that touches `libs/aum-core/`
- Keep `demo-app` minimal — just enough to test aum-core components
- Document breaking changes clearly in commit messages

### DON'T:
- Don't edit the `aum-core-dist` branch directly
- Don't make `libs/aum-core/` changes inside application repositories
- Don't skip the publish script — app repos won't see your changes
- Don't add app-specific or demo-specific code inside `libs/aum-core/`

## Troubleshooting

### Forgot to run the publish script
```bash
npm run publish-core
```

### `aum-core-dist` is out of sync with `main`
```bash
# Delete and recreate
git branch -D aum-core-dist
git push origin --delete aum-core-dist
npm run publish-core
```

### Script fails with uncommitted changes
```bash
git status
git add libs/aum-core/
git commit -m "your message"
npm run publish-core
```

### App repo has merge conflicts after pulling
Resolve conflicts manually, then:
```bash
git add .
git commit -m "chore: resolve aum-core update conflicts"
```

## Creating New Apps

### Option 1: Clone and Customize (Recommended)

```bash
# 1. Clone aum-core as the starting point
git clone https://github.com/rawatanimesh/aum-core.git my-new-app
cd my-new-app

# 2. Customize
#    - Update name/version in package.json
#    - Update app name/routes in apps/demo-app/
#    - Build your feature modules in libs/modules/

# 3. Set up subtree to receive future aum-core updates
./scripts/initialize-core-subtree.sh
```

### Option 2: Start Fresh

```bash
# Create a new NX workspace
npx create-nx-workspace@latest my-new-app
cd my-new-app

# Add aum-core
git remote add aum-core https://github.com/rawatanimesh/aum-core.git
git fetch aum-core
git subtree add --prefix=libs/aum-core aum-core aum-core-dist --squash

# Copy the update script
mkdir -p scripts
cp /path/to/aum-core/scripts/update-aum-core.sh scripts/
chmod +x scripts/update-aum-core.sh
```
